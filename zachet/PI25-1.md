
## Общие требования ко всем вариантам
- Интерфейс: только консольный ввод и вывод.
- Программа должна корректно обрабатывать ошибочный ввод пользователя.
- Запрещено использовать графический интерфейс и сторонние игровые библиотеки.



***

## Вариант 1. Крестики-нолики с настройкой поля

### Постановка задачи

Разработать консольную игру «Крестики-нолики». Перед началом игры пользователь выбирает размер поля: 3x3 или 4x4. После этого два игрока по очереди вводят координаты клетки, в которую хотят поставить свой символ. Программа должна отрисовывать поле после каждого хода, определять победу одного из игроков или ничью. Игры такого типа естественно реализуются как терминальные проекты с циклом ввода, выводом поля и проверкой состояния матрицы.

### Функциональные требования

- Выбор размера поля перед началом партии.
- Отображение игрового поля в консоли.
- Ввод координат хода в формате двух чисел.
- Проверка занятости клетки.
- Определение победы по строке, столбцу или диагонали.
- Определение ничьей.
- Возможность начать новую партию после завершения игры.


### Что проверяется

| Критерий | Баллы |
| :-- | --: |
| Поле создается корректного размера | 8 |
| Ходы игроков обрабатываются правильно | 10 |
| Победа и ничья определяются без ошибок | 12 |
| Есть защита от неверного ввода | 10 |
| Код разбит на функции и читаем | 10 |
| Интерфейс понятен проверяющему | 10 |
| **Итого** | **60** |

### Минимум для зачета

- Игра запускается без ошибок.
- Пользователь может полностью сыграть партию.
- Победа и ничья определяются корректно.
- Ошибочный ввод не ломает программу.


### Возможные команды пользователя

- `start` — начать игру.
- `row col` — сделать ход.
- `restart` — начать заново.
- `exit` — выйти.

## Вариант 2. Угадай число с уровнями сложности

### Постановка задачи

Разработать консольную игру, в которой программа загадывает случайное число, а пользователь пытается его угадать. После каждой попытки выводится подсказка: загаданное число больше или меньше. Перед началом игры пользователь выбирает уровень сложности, от которого зависит диапазон чисел и число попыток. Такой проект обычно строится вокруг цикла ввода, случайной генерации и проверки условий завершения игры.

### Функциональные требования

- Выбор уровня сложности (размер диапазона и кол-во попыток).
- Генерация случайного числа в нужном диапазоне.
- Ввод числа пользователем.
- Подсказка «больше» или «меньше».
- Ограничение числа попыток.
- Подсчет результата партии.
- Возможность повторного запуска.


### Что проверяется

| Критерий | Баллы |
| :-- | --: |
| Сложность влияет на диапазон и число попыток | 12 |
| Загаданное число и проверки работают корректно | 12 |
| Подсказки выводятся верно | 10 |
| Ошибочный ввод чисел обрабатывается безопасно | 10 |
| Код имеет понятную структуру | 8 |
| Интерфейс и сообщения понятны | 8 |
| **Итого** | **60** |

### Минимум для зачета

- Пользователь может выбрать сложность.
- Программа корректно сравнивает введенное число с загаданным.
- При победе и проигрыше выводится итог.
- Неверный ввод не вызывает падения программы.


### Возможные команды пользователя

- `1`, `2`, `3`, ... — выбор сложности.
- `123` — попытка угадать.
- `restart` — новая игра.
- `exit` — выход.
